# 今日小账功能介绍

> **今日小账**是一款独立记账 App，面向希望快速记录日常花销、补账事项、预算目标和角色评账反馈的用户。
> 它首先是一个真实可用的记账软件，其次才是带有酒馆 / SillyTavern 角色短评能力的轻量 RP 添头。

---

## 产品定位

今日小账不是聊天软件，也不是单纯的角色卡展示工具。

它的核心目标是让用户能快速记账、回看明细、理解预算、管理待结清事项，并在需要时让导入的角色根据真实账目给出一句短评。

适用场景包括：

- 日常支出、收入、转账记录
- 月预算和分类预算提醒
- 奶茶、游戏、交通等高频消费统计
- 医疗、办卡、预付、报销等待结清事项
- 酒馆角色卡导入后的账后短评
- Android APK 本地使用

## 快速记账

今日小账默认打开“记账”页，因为记账是最高频动作。

基本流程：

- 选择支出、收入或转账
- 点选分类
- 输入金额
- 写备注
- 选择账户或转账账户
- 保存

如果用户没有手动选择分类，备注会自动参与分类判断。

例如：

- “奶茶 18”自动归入餐饮
- “打车 23”自动归入交通
- “工资 8000”自动归入工资

用户手动选择分类后，手动分类优先，不会被备注自动覆盖。

---

## 预算目标

预算目标放在“我的 > 预算目标”里，不会挤占日常记账界面。

支持：

- 本月总预算
- 分类预算
- 本月已花
- 剩余额度
- 超支判断

记账页会直接显示当前预算状态。

例如用户设置本月预算 2000 元，记账时会显示本月已花和剩余金额。角色短评也可以引用这些真实数据，但不能编造用户没有写过的消费目的。

---

## 待结清 / 找补

待结清用于记录不适合直接算作普通支出的事项。

适合：

- 医疗：总价 3000，医保 1500，自费 1500
- 办卡：花 1000，当前余额 172
- 次数卡：买 2 次，用 1 次，剩 1 次
- 报销：先垫付，后续待报销
- 预付：先付款，后续慢慢消耗

待结清事项默认不计入月支出，除非用户手动生成一笔账。

这样可以避免把“已经支付的钱”和“还需要追踪的权益/余额/次数”混在一起。

## 账户、转账和手续费

今日小账内置轻量账户系统。

默认账户：

- 微信
- 银行卡
- 现金
- 其他

普通支出会扣除支付账户余额，普通收入会增加入账账户余额。

转账时可以选择：

- 转出账户
- 转入账户
- 手续费

转账金额不会算作消费支出；如果填写手续费，手续费会自动生成一笔支出。

## 明细与统计

明细页保留独立的时间范围：

- 当天
- 本周
- 当月
- 当年
- 总计

查看方式单独选择：

- 明细列表
- 饼图
- 条形图

统计内容包括：

- 收入
- 支出
- 结余
- 预算剩余
- 待结清数量
- 分类消费金额
- 分类消费次数

AI 月总结只读取这些真实统计，不自动补不存在的账。

## 角色评账

角色评账是附加功能，不改变记账软件的核心定位。

支持：

- 导入 SillyTavern / 酒馆角色卡
- 读取角色名、头像、人设、性格和世界书
- 启用多个角色参与评账
- 每次记账后自动短评
- 单独修改用户信息和全局 Prompt

评账规则：

- 一般只输出一句话
- 不输出编号
- 不写长篇理财报告
- 不虚构地点、人物、路线、剧情
- 可以引用真实账单、预算剩余、分类次数和备注

角色长期偏好放在角色信息里，世界书更适合放全局规则。

---

## API 与模型

今日小账支持 OpenAI 兼容接口。

在“我的 > API 设置”里可以配置：

- 接口地址
- API Key
- 拉取模型
- 测试 API

API Key 保存在本机浏览器数据里，不写进源码仓库。
